작업환경

개발환경

  1. visual studio code를 사용한다.
  2. 마크업 들여쓰기는 TAB 크기 4 로 맞춰 작업한다.

형상관리(SVN) 사용방법

  1. 프로젝트를 받을 파일에서 우클릭 후 'SVN Checkout...'을 눌러줍니다.

  2. 그러면 다음과 같은 창이 뜹니다. 여기서 'URL of directory'칸에 받아올 프로젝트의 url 을 넣고 'OK' 버튼을 클릭한다.

  3. 참고로 여기서 URL of repository 칸 오른쪽에 '...' 을 클릭하면 받아올 프로젝트의 구조를 볼 수 있습니다.

  4. Checkout Finished 창이 뜨고 'OK'버튼을 클릭한다.

디렉토리 구조

  1. 해당 프로젝트명을 상위 폴더로 한다.
  2. 스타일에 관련된 폴더(css, images, font 등)는 assets 폴더를 상위에 두고 사용하거나, 각각의 폴더를 독립적으로 사용 하는 것을 권장한다. assets > css, images, font || css, images, font (독립적 폴더)

다국어 분류는 상위 폴더를 국가 별로 감싸서 분류 (KO > html, css, js ... && EN > html, css, js ...) 하거나, 현 폴더 내부 에서 분류 (html > ko, en ,cn) 한다.

CSS 파일역할

속성 선언 순서표
파일명 역할 항목
reset.css 초기화 폰트 초기화, 도큐먼트 초기화
font.css 사용할 서체 선언 서체 정의 및 호출
common.css 사이트 공통 선언 (공통) 폰트컬러 예약어, 여백관련 예약어, 버튼컬러예약어, 타이틀 정의, 링크스타일, 폼스타일, 텍스트아레아, 버튼, 탭기본, 아이콘기본 등 주석으로 항목 분류
layout.css 레이아웃 공통레이아웃, 서브공통, 서브개별, 그리드 시스템
main.css 메인 고유 스타일 공통, 섹션, 메인 컨텐츠

mobile에서는 각 파일명 끝에 _m으로 사용한다.(common_m.css, layout_m.css 등)
