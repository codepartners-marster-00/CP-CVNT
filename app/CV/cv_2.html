Naming rules

공통규칙

  1. 네이밍의 첫 시작에 숫자, 특수문자, 대문자의 사용은 지양하며, 영문 소문자, 숫자, underscore(_)로 작성한다
  2. 네이밍은 '형태_의미_상태' 순서로 조합하며, 3단계를 넘어가지 않도록 권장한다. btn_apply_on, box_news, box_reply_open...
  3. 네이밍 정의 시 prefix, subfix, suffix를 최대한 활용한다.
  4. 네이밍 정의 시 임의로 축약하지 않는다.

hyphen(-)을 사용 할 경우는 단계(depth)를 나타낼때 주로 사용하며, 단어간의 연결은 underscore(_)를 사용하는 것을 권장한다.

id 및 class 네이밍 규칙

PC HTML 기본 템플릿 구조

id/class

  1. id는 camelcase 방식으로 하며, class는 underscore 방식으로 사용한다.
  2. id는 화면에서의 고유 기능을 명시하도록 네이밍 한다. id="btnGnbSearch"
  3. class는 요소 기능을 표현하도록 네이밍 한다.
  4. 레이아웃을 제외한 id는 스타일을 지정하지 않는다.
  5. id는 문서 내 한 번만 사용되며, id, class 네이밍은 가급적 같지 않게 네이밍 한다.

class 네이밍 확장

  1. 기본형 class에 종속되어 여백, 색깔, 행간 등의 몇 가지 속성을 부여하고자 할 때는 종속 확장 class로 사용한다. 기본형 - mykin_list / 확장형 - mykin_list_v1, mykin_list_v2...
  2. 기본형 class의 변형이 타입으로 분류할 만큼 클 경우엔 독립 확장 class로 사용한다. 기본형 - mykin_list / 확장형 - mykin_list2, mykin_list3

이미지 네이밍 규칙

네이밍 규칙 및 조합

  1. 같은 분류의 이미지가 두 개 이상이면 파일 이름 마지막에 숫자를 추가하여 구분한다.
  2. 이미지 네이밍은 이미지 확장자와 관계 없이 순차적으로 적용한다. bu_dot1.gif,  &nbbu_dot2.jpg,  bu_dot3.png
  3. 임시 이미지는 tmp_를 조합한다. tmp_

파일 및 디렉토리 네이밍 규칙

네이밍 규칙 및 조합

  1. HTML파일은 '메뉴이름_의미_상태' 순서로 조합한다. news_nboard_view.html
  2. CSS 파일은 '서비스이름'을 맨 앞으로 하여 조합한다. news_home.css

prefix/subfix/suffix 정의

prefix

  1. 접두사를 의미하는 것으로, 앞부분에 사용하며, 주로 형태를 나타내는 데 사용된다.tab_notice, tbl_product
  2. 기본 prefix는 형태별로 통일한다.
  3. 서브 prefix가 필요한 부분은 underscore로 구분하여 표기한다. line / line_dot
  4. prefix 2개 이상 중복 시 기능적인 prefix가 맨 앞부분에 위치할 수 있도록 한다.
  5. 단독사용 클래스는 지양하며, prefix를 조합한 형태로 작성한다. 예외: fst, lst, hide, on
  6. on 클래스인 경우, 단일 특정요소에 상태변화 클래스를 적용한다면 suffix로서 적용한다.
